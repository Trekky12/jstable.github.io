General

sortable

Default: true
Type: Boolean

enable/disable sorting of the columns

searchable

Default: true
Type: Boolean

enable/disable searching

Columns

columns

Type: Array with column objects

the columns option can hold multiple column objects

Column object properties

select

Type: Integer or Array

select one or multiple columns for the following options

searchable

Type: Boolean

enable/disable searching for the columns in select

sortable

Type: Boolean

enable/disable sorting for the columns in select

sort

Type: asc" or "desc"

When the sort option is set, the table is initial sorted by this column with the defined sorting direction.
The sorting is only applied if the sortable option set to true
This property can only be applied to single columns, because the table can only be sorted by one column at the time.

render

Type: function (cell, idx)

the rendering function for the selected column(s) with the cell and the cell index as argument.
The function has to return the rendering for this column(s).

Example

In the following example the first column (0) is sortable and the table is initial sorted by this column in "asc" direction.
This column is also searchable and rendered with moment.js in the date format 'DD.MM.YYYY'.

The second (1) and third (2) columns are not sortable and not searchable.
The table cells have a data-attribute "data-link" which is used to render the table with a hyperlink.

columns: [
            {   
                select: 0,
                sortable: true,
                sort: "asc",
                searchable: true,
                render: function (cell, idx) {
                    let data = cell.innerHTML;
                    return moment(data).format('DD.MM.YYYY');
                }
            },
            {   
                select: [1, 2],
                sortable: false,
                searchable: false,
                render: function (cell, idx) {
                    let data = cell.innerHTML;
                    let link = cell.dataset.link;
                    return '<a href="' + link + '">' + data + '</a>';
                }
            }
]

Data-Attributes

It is possible to define data-attributes on individual columns by applying them to the table header cell.

Sorting

data-sortable

Type: Boolean

enable/disable sorting of the column

  • a global definition of the sortable attribute of the table can be overwritten
    by a "data-sortable" attribute on the table header or
    by a custom column definitions attribute "sortable"
  • a custom column definitions attribute "sortable" can be overwritten
    by a "data-sortable" attribute on the table header
  • a "data-sortable" attribute on the table header cannot be overwritten

data-sort

Type: asc" or "desc"

sort the table initially by this column in the defined direction

  • a custom column definitions attribute is overwritten by a "data-sort" attribute
  • since sorting is only supported for one column at the same time the last defined column is used

Example

In the following example the table is sortable and should be initially sorted by the third column.

The data-attribute "data-sortable" disabled the sorting for the first column and the data-attribute "data-sort" changes the initial sorting to the second column.

HTML

<table id="example2">
    <thead>
        <tr>
            <th data-sortable="false">Name</th>
            <th data-sort="asc">Country</th>
            <th>Date</th>
            <th>Number</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Norman Small</td>
            <td>Tokelau</td>
            <td>2020-02-01 07:22:40</td>
            <td>8243</td>
        </tr>
    </tbody>
</table>

JavaScript

new JSTable("#example2", {
        sortable: true,
        columns: [
            {
                select: 3,
                sortable: true,
                sort: "asc"
            }
        ]
    }
);

Dropdown

perPage

Default: 5
Type: Integer

the maximum number of elements per page

perPageSelect

Default: [5, 10, 15, 20, 25]
Type: Array

the per page options in the dropdown as a list of integers. false hides the dropdown entirely.

Pagination

nextPrev

Default: true
Type: Boolean

enable/disable the previous/next buttons

prevText

Default: "&lsaquo;"
Type: String

content of the previous button

nextText

Default: "&rsaquo;"
Type: String

content of the next button

firstLast

Default: false
Type: Boolean

enable/disable the buttons to go to first and last page

firstText

Default: "&laquo;"
Type: String

content of the go to first page button

lastText

Default: "&raquo;"
Type: String

content of the go to last page button

truncatePager

Default: true
Type: Boolean

truncate the page links

pagerDelta

Default: 2
Type: Integer

when truncating the page links show always pagerDelta elements before and after the current page

ellipsisText

Default: "&hellip;"
Type: String

content of item which indicates the truncated links

Labels

labels

Type: object
Default: 
labels: {
    placeholder: "Search...",
    perPage: "entries per page",
    noRows: "No entries found",
    info: "Showing {start} to {end} of {rows} entries",
    loading: "Loading...",
    infoFiltered: "Showing {start} to {end} of {rows} entries (filtered from {rowsTotal} entries)"
}

customization of display text messages

Propery Element Variables
placeholder placeholder of the search input field
perPage label of the elements per page selection
noRows message text when no elements where found after doing a search
info text for the {info} layout element {start}: the first element on this page
{end}: the last element on this page
{page}: the current page number
{pages}: the total page number
{rows}: the total number of found elements (eventually filtered)
{rowsTotal}: the total number of elements
loading loading text
infoFiltered text for the {info} layout element when the table is filtered see info

Layout

layout

Type: object
Default: 
layout: {
    top: "{select}{search}",
    bottom: "{info}{pager}"
}

customization of the elements in the top and bottom containers

Variable Element
{select} the element per page selection dropdown
{search} the search field
{info} the info message
{pager} the pagination

Notices

  • It is not advised to use the {search} and {select} elements more than once, since they are not synchronized and work independent
  • The {info} and {pager} elements can only be used once.
  • When using only one element this element is always display on the left.
    It is not possible to display a single element on the right aligned.
  • The element {select} depends on the value of perPageSelect. When the perPageSelect option is false the container is not displayed.
  • The element {search} depends on the value of searchable. When the searchable option is false the container is not displayed.

Server Side Options

For large datasets it is possible to do the sorting, searching and pagination on the server.

The first table page needs to be available in plain HTML.
For the various actions the server is contacted and the table is replaced with the returning data from the server.

serverSide

Default: false
Type: Boolean

should the table data be retrieved from the server?

deferLoading

Default: null
Type: Integer

total data count

ajax

Default: null
Type: String

the URL for retrieving the data

ajaxParams

Default: {}
Type: object

additional parameters for the server

Default Query Parameters

Parameter Value
searchQuery the search text
sortColumn the index of the to sorted column
sortDirection the sorting direction (asc or desc)
start the data offset
length the length of the returning data

Server Response

Parameter Value
recordsTotal total data count
recordsFiltered filtered data count
data the table data which should be displayed as a two dimensional JSON array

Query Parameters

It is possible to append URL query parameters when searching or navigating with the current selected value.
Additionally these parameters are parsed and applied when loading the current page.

addQueryParams

Default: true
Type: Boolean

enable/disable query parameter appending to URL

queryParams

Type: object
Default: 
queryParams: {
    page: 'page',
    search: 'search'
}

the name of the query parameters to append/parse
